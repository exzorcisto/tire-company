# Лабораторная работа №3: Интеграция Redux Toolkit в Sibirea Tires

## Цель работы
Изучение и внедрение инструмента **Redux Toolkit** для управления глобальным состоянием приложения. Основная задача — разграничить серверное состояние (управляемое React Query) и клиентское состояние (авторизация, корзина заявок).

## Стек технологий
* **Redux Toolkit** — управление логикой стейта.
* **React-Redux** — связка Redux с компонентами React.
* **Immer** — встроенная библиотека для работы с иммутабельностью.

## Реализованная архитектура (Folder Structure)
Логика Redux вынесена в отдельную директорию `src/store/`:
* `index.js` — конфигурация Store с использованием `configureStore`.
* `slices/` — отдельные файлы-срезы для каждой логической части:
  * `authSlice.js` — управление состоянием пользователя (вход, выход).
  * `cartSlice.js` — управление корзиной (добавление/удаление заявок на шины).

## Основные возможности
1. **Глобальный доступ**: Данные о количестве заявок доступны в `Header.jsx` и `Catalog.jsx` без проброса пропсов.
2. **Мемоизация**: В `cartSlice.js` реализован `createSelector` для эффективного подсчета общего количества товаров в корзине.
3. **Persist (Локальное хранение)**: Реализовано сохранение данных пользователя в `localStorage` через редьюсеры `authSlice`.
4. **Безопасные мутации**: Благодаря интеграции библиотеки **Immer**, в редьюсерах используется интуитивно понятный синтаксис (например, `.push()`), при этом сохраняется принцип иммутабельности React.

## Как это работает
* **Интеграция**: Приложение обернуто в `<Provider store={store}>` в файле `main.jsx`.
* **Dispatch**: Кнопка "В ЗАЯВКУ" в каталоге отправляет экшен `addToCart`.
* **Selector**: Компонент шапки подписывается на изменения через `useSelector` и мгновенно обновляет счетчик.

## Отладка
Для мониторинга состояния используется расширение **Redux DevTools**, позволяющее отслеживать историю каждого экшена и текущее дерево стейта.
